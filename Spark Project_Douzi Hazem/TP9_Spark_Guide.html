📚 1. Concepts de base : Spark & RDD

Qu'est-ce que Spark ?

Apache Spark est un moteur de traitement distribué pour les données massives (Big Data). Il est :

  • ⚡ Rapide : calcule en mémoire (jusqu'à 100× plus rapide que Hadoop MapReduce)
  • 🐍 Polyglotte : disponible en Python (PySpark), Scala, Java, R
  • 📦 Riche : RDD, DataFrame, MLlib (machine learning), Streaming, GraphX

Qu'est-ce qu'un RDD ?

RDD = Resilient Distributed Dataset (jeu de données distribué et résilient). C'est la structure de données fondamentale de Spark : une collection d'éléments répartie sur plusieurs machines (ou cœurs CPU).

TypeDescriptionExemples
Transformation Crée un nouveau RDD à partir d'un autre (évaluation paresseuse) map, filter, flatMap, reduceByKey, groupByKey
Action Déclenche le calcul et retourne un résultat collect, count, reduce, take, saveAsTextFile
🎯 Le pattern MapReduce :
  1. Map : transformer chaque élément (ex : x → (x, 1))
  2. Shuffle : regrouper par clé
  3. Reduce : agréger les valeurs (ex : somme, max, moyenne)

🟢 Exercice 1 — Word Count avec PySpark

Objectif : compter le nombre d'occurrences de chaque mot dans un texte. C'est le « Hello World » de Spark.

1Créer un fichier wordcount.py dans votre dossier projet.
2Importer Spark et créer le contexte : 
from pyspark import SparkContext, SparkConf

conf = SparkConf().setAppName("WordCount").setMaster("local[*]")
sc = SparkContext(conf=conf)
sc.setLogLevel("ERROR")

📌 local[*] signifie : « utilise tous les cœurs CPU de ma machine ».

3Créer le texte et le mettre dans un RDD : 
text = [
    "Apache Spark is a fast and general engine for big data processing",
    "Spark provides high level APIs in Java Scala Python and R",
    "Spark is fast Spark is powerful Spark is easy to use",
    "Big data processing with Spark is awesome"
]
rdd = sc.parallelize(text)
4Pipeline MapReduce : 
word_counts = (rdd
    .flatMap(lambda line: line.lower().split(" "))   # découper en mots
    .map(lambda word: (word, 1))                       # (mot, 1)
    .reduceByKey(lambda a, b: a + b))                  # sommer par mot

Détail des transformations :

  • flatMap : découpe chaque ligne en mots et "aplatit" en une seule liste
  • map : transforme chaque mot en couple (mot, 1)
  • reduceByKey : pour chaque mot identique, additionne les 1
5Exécuter et afficher : 
results = word_counts.sortBy(lambda x: -x[1]).collect()
for word, count in results:
    print(f"{word:20s} : {count}")
sc.stop()

(a) Maximum dépensé par un client ayant noté « awesome »

1L'expression de l'énoncé : 
result = (dataset
  .map(lambda (id, product, spent, rating): (rating, spent))
  .filter(lambda (rating, spent): rating == 'awesome')
  .map(lambda (rating, spent): (rating, spent))
  .reduce(lambda first, second: max(first, second)))

📝 Signification du résultat

Le résultat de cette expression est la plus grosse somme dépensée par un client ayant noté un produit "awesome".

Avec notre dataset : les clients « awesome » sont les ids 1, 3, 5, 6, 9 avec montants 2, 1, 8, 1, 3. Le maximum est donc 8 € (le client 5 qui a acheté des macadamia cookies).

« Cela permet d'identifier le client le plus dépensier parmi ceux qui sont totalement satisfaits. »

(b) Calculs avancés sur les cookies

📥 Charger le CSV

from pyspark.sql import SparkSession

spark = SparkSession.builder.appName("Cookies").master("local[*]").getOrCreate()
rdd = spark.read.csv("customercookies.csv", header=False).rdd

(b)(i) — Nombre de notes « awesome » par produit

awesome_count = (rdd
    .filter(lambda r: r[3] == 'awesome')
    .map(lambda r: (r[1], 1))
    .reduceByKey(lambda a, b: a + b))

total_spent = (rdd

(b)(iii) — Moyenne dépensée par produit

💡 Astuce : on combine total + count avec mapValues puis on divise.

avg_spent = (rdd
    .filter(lambda r: r[3] == 'awesome')
    .map(lambda r: (r[1], (float(r[2]), 1)))
    .reduceByKey(lambda a, b: (a[0]+b[0], a[1]+b[1]))

🔵 Exercice 3 — k-Means avec Spark

Objectif : implémenter manuellement l'algorithme de clustering k-Means avec PySpark.

🧠 Comment fonctionne k-Means ?

  1. 📍 Choisir k centroïdes initiaux (au hasard parmi les points)
  2. 🎯 Affecter chaque point au centroïde le plus proche → ça crée k clusters
  3. 🔄 Recalculer chaque centroïde = moyenne des points de son cluster
  4. 🔁 Répéter 2-3 jusqu'à convergence (les centroïdes ne bougent plus)
🎯 Les 2 fonctions à implémenter (TODO de l'énoncé) :
  • assign_to_centroid(point, centroids) → retourne (index_centroide, point)
  • calculate_new_centroid(points) → retourne le barycentre des points

Étape 1 : générer le dataset blobs.csv

Utiliser sklearn.datasets.make_blobs :

from sklearn.datasets import make_blobs
import numpy as np

X, y = make_blobs(n_samples=300, centers=3, n_features=2,
                  cluster_std=1.0, random_state=42)
np.savetxt("blobs.csv", X, delimiter=",", fmt="%.6f")

Cela crée 300 points organisés en 3 « blobs » (groupes) qu'on va devoir retrouver.

Étape 2 : implémenter assign_to_centroid

def assign_to_centroid(point, centroids):
    # Distance euclidienne entre le point et chaque centroïde
    distances = [np.linalg.norm(point - c) for c in centroids]
    # Index du centroïde le plus proche
    closest_index = int(np.argmin(distances))
    return (closest_index, point)

Étape 3 : implémenter calculate_new_centroid

def calculate_new_centroid(points):
    points_array = np.array(list(points))
    return points_array.mean(axis=0)   # moyenne des coordonnées

Étape 4 : la boucle principale

def kmeans(rdd_points, k=3, max_iter=20):
    centroids = rdd_points.takeSample(False, k, seed=42)

    for i in range(max_iter):
        clusters = rdd_points.map(lambda p: assign_to_centroid(p, centroids))
        new_centroids = (clusters
                         .groupByKey()
                         .mapValues(calculate_new_centroid)
                         .collect())
        # ... vérifier convergence + mise à jour ...
    return centroids
